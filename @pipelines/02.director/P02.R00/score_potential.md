# 특허성 잠재력 채점 (Patentability Potential)

발명이 *특허로 등록될 가능성* 을 평가. 종래기술 대비 차별점·효과 명시 정도. 2 sub-항목:

1. **differentiation** — 종래기술 대비 *차별점* (= 기존과 무엇이 다른가)
2. **effect** — *효과* 의 구체성 (= 발명으로 얻는 결과·이점)

input:
- `stack_differentiation`: 차별점 설명 (string)
- `stack_effects`: 효과 list

## Rubric (각 sub-항목, 0.0 ~ 1.0)

### differentiation

| 점수 | 기준 |
|---|---|
| 0.00 | 차별점 언급 없음 (empty) |
| 0.25 | "더 좋다", "편하다" 등 추상적 차이만 |
| 0.50 | 종래기술 1 개와 구체적 비교 (e.g., "기존 X 와 달리 Y") |
| 0.75 | 다수 종래기술 + 비교 + 기술적 차이 명시 |
| 1.00 | 종래기술 분석 + 핵심 기술적 차이 + 진보성 시사 |

### effect

| 점수 | 기준 |
|---|---|
| 0.00 | 효과 언급 없음 (빈 list) |
| 0.25 | 1 개 추상적 효과 (e.g., ["편리함"]) |
| 0.50 | 2-3 개 구체적 효과 (e.g., ["설치 시간 50% 단축"]) |
| 0.75 | 다중 효과 + 정량적 측정 가능 + 사용자 가치 명확 |
| 1.00 | 정량/정성 효과 모두 + 산업적 활용 + 경제적 가치 |

## 출력 (JSON only)

```json
{
  "differentiation": 0.60,
  "effect": 0.50,
  "rationale": "왜 이 점수인지 한국어 2-4 문장 (각 sub 점수 근거 + 종래기술 비교의 부족 또는 효과의 정량화 여부)"
}
```

- 점수는 0.0 ~ 1.0, 소수점 2자리.
- 5 단계 rubric 은 가이드 — 연속 값 허용.
- `rationale` 은 20자 이상.

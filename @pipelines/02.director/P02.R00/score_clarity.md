# 아이디어 구체도 채점 (Concept Clarity)

발명의 *기본 정체성* 을 평가. 2 sub-항목:

1. **purpose** — 발명의 *목적* 명시 정도 (= 어떤 문제·필요·응용 분야)
2. **components** — *핵심 구성요소* 식별 정도 (= 발명을 이루는 필수 element)

input:
- `stack_purpose`: 현재까지 정리된 발명 목적 (string)
- `stack_components`: 현재까지 정리된 핵심 구성요소 (list of string)

## Rubric (각 sub-항목, 0.0 ~ 1.0)

### purpose

| 점수 | 기준 |
|---|---|
| 0.00 | 발명 목적 언급 없음 (empty 또는 무관) |
| 0.25 | 일반적 영역만 (e.g., "스마트 디바이스 만들고 싶어") |
| 0.50 | 구체적 문제 + 대상 사용자/분야 (e.g., "거치대가 자꾸 떨어지는 문제를 해결") |
| 0.75 | 문제 + 해결 방향 명시 (e.g., "거치대가 자꾸 떨어지는 문제를 마찰력 증가로 해결") |
| 1.00 | 문제 + 해결 + 응용 영역·차별 의도 모두 명시 |

### components

| 점수 | 기준 |
|---|---|
| 0.00 | 구성요소 식별 없음 (빈 list 또는 무관) |
| 0.25 | 1 개만 식별, 추상적 (e.g., ["거치대"]) |
| 0.50 | 2-3 개 핵심 element + 일부 구체화 |
| 0.75 | 3-5 개 element + 각자 역할 명시 |
| 1.00 | 모든 필수 element + 상호작용·계층 구조 명확 |

## 출력 (JSON only)

```json
{
  "purpose": 0.65,
  "components": 0.50,
  "rationale": "왜 이 점수인지 한국어 2-4 문장 (각 sub 점수 근거 + 부족한 부분 + 다음 사이클에서 보완될 여지)"
}
```

- 점수는 0.0 ~ 1.0, 소수점 2자리 (multipleOf 0.01).
- 5 단계 rubric 은 *가이드* — 0.62, 0.83 같은 연속 값도 자연스러우면 허용 (LLM 의 reasoning 우선).
- `rationale` 은 20자 이상.

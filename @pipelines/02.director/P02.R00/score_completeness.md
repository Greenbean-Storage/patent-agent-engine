# 설명의 완전성 채점 (Description Completeness)

발명이 *작동하는 방식* 을 사용자가 얼마나 설명했는지 평가. 3 sub-항목:

1. **sequence** — *동작 순서* 의 명료성 (= 단계별 흐름)
2. **causality** — *인과 관계* 의 명료성 (= "X 가 일어나면 Y")
3. **embodiment** — *실시예* 의 풍부함 (= 구체적 사용 사례)

input:
- `stack_sequence`: 동작 순서 list
- `stack_causality`: 인과 관계 list
- `stack_embodiments`: 실시예 list

## Rubric (각 sub-항목, 0.0 ~ 1.0)

### sequence

| 점수 | 기준 |
|---|---|
| 0.00 | 동작 순서 언급 없음 (빈 list) |
| 0.25 | 1-2 단계만 (e.g., ["거치한다"]) |
| 0.50 | 3-5 단계 + 시작/끝 명확 |
| 0.75 | 5+ 단계 + 분기·조건 일부 명시 |
| 1.00 | 전체 lifecycle + 분기·예외 처리 완전 |

### causality

| 점수 | 기준 |
|---|---|
| 0.00 | 인과 관계 언급 없음 |
| 0.25 | 단순 트리거-결과 1 개 |
| 0.50 | 2-3 개 메커니즘 + 원리 시사 |
| 0.75 | 다중 원인-결과 + 조건 명시 (X+Y 면 Z 등) |
| 1.00 | 물리·전기·논리적 원리 명확 + 예측 가능 |

### embodiment

| 점수 | 기준 |
|---|---|
| 0.00 | 실시예 없음 (빈 list) |
| 0.25 | 1 개 추상적 (e.g., ["어디서나 사용"]) |
| 0.50 | 2-3 개 + 사용 맥락 구체 (e.g., ["차량 내부", "실내 책상"]) |
| 0.75 | 3-5 개 + 환경·조건·변형 다양 |
| 1.00 | 5+ 개 + 다양한 형태·재질·크기·사용 시나리오 |

## 출력 (JSON only)

```json
{
  "sequence": 0.50,
  "causality": 0.40,
  "embodiment": 0.30,
  "rationale": "왜 이 점수인지 한국어 2-4 문장 (각 sub 점수 근거 + 어떤 정보가 더 필요한지)"
}
```

- 점수는 0.0 ~ 1.0, 소수점 2자리.
- 5 단계 rubric 은 가이드 — 연속 값 허용.
- `rationale` 은 20자 이상.
